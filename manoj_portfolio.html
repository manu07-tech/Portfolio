Things I've Built

Smart Road Accident Prediction
P-01

Road accidents cause thousands of fatalities annually due to unidentified risk patterns. This project applies data mining to predict high-risk accident zones and support preventive action by road safety authorities.

PythonData MiningPandasMatplotlibClassificationDashboards
Key Features
  • Processes historical accident datasets to identify contributing risk factors such as weather, speed, and time of day
  • Applies classification algorithms to predict accident likelihood at specific road segments
  • Visualises risk patterns through graphs and summary dashboards for decision-makers
  • Generates safety insights to support road planning and infrastructure decisions
What I Learned
This project deepened my understanding of the full data pipeline — from CSV ingestion to model output visualisation. I learned how to structure a data workflow, interpret model metrics, and communicate findings clearly to non-technical audiences.
Student Management System
P-02

Educational institutions often rely on error-prone spreadsheets to manage student records. This console-based Java application provides a CRUD-driven backend using JDBC and MySQL to manage student data reliably and at scale.

Core JavaJDBCMySQLOOPCollectionsException Handling
Key Features
  • Add, update, delete, and search student records via a menu-driven console interface
  • Persistent storage via MySQL; all DB operations are wrapped with proper exception handling
  • Uses ArrayList and HashMap for in-memory caching of frequently accessed queries
  • Modular design with separate classes for StudentDAO, DBConnection, and main controller
What I Learned
Building this gave me hands-on experience with the DAO pattern and JDBC lifecycle. I understood why separating concerns into distinct layers makes code far easier to test, extend, and maintain over time.
Task Tracker REST API
P-03

A lightweight Spring Boot REST API for a personal task management application. Demonstrates back-end API design skills — endpoints, request/response mapping, HTTP status codes, and clean project structure.

Spring BootREST APIJavaPostmanMavenJSON
Key Features
  • Full CRUD via RESTful endpoints: GET /tasks, POST /tasks, PUT /tasks/{id}, DELETE /tasks/{id}
  • Proper HTTP status codes (200, 201, 404, 400) and structured JSON response bodies
  • Request validation and error handling using @RestController and @RequestBody annotations
  • Endpoints tested and documented via Postman collections
What I Learned
This project gave me confidence designing API contracts before writing code — a professional habit I now apply to all back-end work. I understood how Spring Boot abstracts boilerplate and how annotations work under the hood.
